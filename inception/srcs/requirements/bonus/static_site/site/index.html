Inception

Progetto di System Administration con Docker

Inception è un progetto basato sulla comprensione e sull'uso di Docker. Nello specifico, di Docker CLI.

Docker e Docker CLI

Docker è una piattaforma di containerizzazione che consente di creare, distribuire ed eseguire applicazioni in ambienti isolati chiamati container. La Docker CLI (Command Line Interface) è lo strumento principale per interagire con Docker, permettendo di gestire immagini, container, reti e volumi attraverso comandi testuali.

I container Docker offrono vantaggi significativi rispetto alla virtualizzazione tradizionale: sono più leggeri, si avviano più rapidamente e condividono il kernel del sistema operativo host. Docker CLI fornisce comandi essenziali come docker build per creare immagini, docker run per eseguire container, e docker-compose per orchestrare servizi multi-container.

Il progetto si compone di due parti fondamentali: mandatory e bonus.

Parte Mandatory

La parte mandatory prevede l'integrazione nel progetto, di 3 servizi:

  • NGINX
  • MariaDB
  • WordPress

Panoramica dei Servizi

NGINX

NGINX è un server web ad alte prestazioni che funziona anche come reverse proxy, load balancer e cache HTTP. Nel progetto Inception, NGINX serve come punto di ingresso per tutte le richieste web, gestendo il traffico HTTPS e instradando le richieste verso il container WordPress. La sua configurazione deve includere certificati SSL/TLS per garantire connessioni sicure.

MariaDB

MariaDB è un sistema di gestione di database relazionale open-source, fork di MySQL. Nel contesto del progetto, MariaDB fornisce il database backend per WordPress, memorizzando contenuti, utenti, configurazioni e metadati. È necessario configurare utenti specifici, database dedicati e impostare le corrette autorizzazioni per garantire la sicurezza e il funzionamento del sistema.

WordPress

WordPress è un sistema di gestione dei contenuti (CMS) che permette di creare e gestire siti web dinamici. Nel progetto Inception, WordPress è configurato per utilizzare PHP-FPM (FastCGI Process Manager) per elaborare le richieste PHP in modo efficiente. Deve essere configurato per connettersi al database MariaDB e servire contenuti attraverso NGINX.

Requisiti

Requisiti Generali

  • Ogni servizio deve essere containerizzato in un container Docker separato
  • Utilizzo di Dockerfile personalizzati per ogni servizio (vietato l'uso di immagini precostituite)
  • Orchestrazione dei servizi tramite docker-compose
  • Configurazione di una rete Docker personalizzata per la comunicazione tra container
  • Implementazione di volumi per la persistenza dei dati
  • Utilizzo di variabili d'ambiente per le configurazioni sensibili

Requisiti NGINX

  • Configurazione SSL/TLS obbligatoria (porta 443)
  • Supporto per TLSv1.2 o TLSv1.3
  • Redirect automatico da HTTP a HTTPS
  • Configurazione come reverse proxy per WordPress
  • Gestione delle richieste statiche e dinamiche

Requisiti WordPress

  • Installazione e configurazione di PHP-FPM
  • Configurazione del file wp-config.php
  • Connessione al database MariaDB
  • Creazione di almeno due utenti (amministratore e utente normale)
  • Configurazione dei permessi dei file

Requisiti MariaDB

  • Creazione di un database dedicato per WordPress
  • Configurazione dell'utente root
  • Creazione di un utente specifico per WordPress con privilegi limitati
  • Configurazione per accettare connessioni dalla rete Docker
  • Implementazione di meccanismi di backup e recovery

Parte Bonus

La parte bonus, come indicato anche dal subject, prevede un'implementazione non particolarmente impegnativa, in quanto il progetto in sé ha già fornito molti concetti da acquisire e l'implementazione di ulteriori servizi non è vista tanto come l'uso del servizio in particolare, quanto l'abilità da parte dell'implementatore di padroneggiare correttamente Docker, container e la comunicazione che avviene tra essi.

Tra i servizi implementabili nella parte bonus, troviamo:

  • Redis
  • FTP
  • Adminer
  • Un server statico
  • Un servizio aggiuntivo a scelta

Panoramica dei Servizi Bonus

Redis

Redis è un database in-memory ad alte prestazioni utilizzato principalmente come cache. Nel contesto di WordPress, Redis può essere implementato come object cache per migliorare significativamente le prestazioni del sito, riducendo il carico sul database MariaDB. La sua implementazione richiede l'installazione del plugin Redis Object Cache e la configurazione appropriata per permettere a WordPress di utilizzare Redis come sistema di caching.

FTP Server

L'implementazione di un server FTP (tipicamente vsftpd) permette di gestire i file del sito WordPress in modo più flessibile. Questo servizio consente agli utenti di caricare, modificare e gestire i file del sito attraverso client FTP. La configurazione deve includere utenti dedicati, chroot jail per la sicurezza e integrazione con i volumi condivisi con gli altri servizi.

Adminer

Adminer è un'interfaccia web per la gestione di database, alternativa più leggera e moderna a phpMyAdmin. Fornisce un'interfaccia grafica per gestire il database MariaDB, permettendo di eseguire query, modificare strutture delle tabelle e amministrare utenti e privilegi. La sua implementazione richiede la configurazione di PHP e l'integrazione con il proxy NGINX.

Server Statico

Un server statico può essere implementato utilizzando generatori di siti statici come Hugo, Jekyll, o semplicemente servendo file HTML (eventualmente css, js) statici. Questo servizio dimostra la capacità di integrare diversi tipi di contenuti web nell'architettura Docker e può essere utilizzato per documentazione, portfolio personale o landing page. La configurazione richiede l'integrazione con NGINX per il routing appropriato.

Servizio aggiuntivo

Il servizio aggiuntivo a scelta offre la libertà di implementare qualsiasi tecnologia che dimostri la padronanza di Docker. Esempi possibili includono: monitoring con Prometheus/Grafana, message queue con RabbitMQ, search engine con Elasticsearch, o servizi di autenticazione. La scelta dipende dall'interesse personale e dagli obiettivi di apprendimento.

Considerazioni sui requisiti bonus

Ogni servizio, come per la parte mandatory deve rispettare i requisiti generali di buona implementazione, ma in questo caso non vi sono regole particolarmente mirate. Essendo un bonus, viene data maggiore margine di implementazione al cadet.

I principi guida per l'implementazione dei servizi bonus includono:

  • Mantenimento dell'architettura containerizzata
  • Integrazione coerente con l'infrastruttura esistente
  • Configurazione sicura e best practices
  • Documentazione appropriata dell'implementazione
  • Testing e verifica del funzionamento

Conclusione

Il progetto Inception rappresenta un'opportunità completa per apprendere i concetti fondamentali di Docker e dell'orchestrazione di container. Attraverso l'implementazione di servizi reali in un ambiente containerizzato, si acquisiscono competenze pratiche di system administration, networking, sicurezza e deployment che sono essenziali nel panorama tecnologico moderno.

La combinazione di servizi mandatory e bonus offre un equilibrio tra requisiti strutturati e libertà creativa, permettendo di esplorare diverse tecnologie mentre si mantiene focus sui principi fondamentali di containerizzazione e orchestrazione.
